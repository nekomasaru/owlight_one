\## 概要



定期的な1on1や人事評価に向け、ユーザーが自身の成果や悩みを言語化するための入力フォームを実装します。

単なるテキスト入力ではなく、「今月のMVPは誰でしたか？」「何に一番苦労しましたか？」といったコーチング要素のある質問項目（チェックリスト）を提示し、ユーザーの振り返りをサポートします。また、書きかけの内容を保存できる「下書き保存」機能を備えます。







\## このスキルが前提とするもの



\- `evaluation-api-submit` スキルにより、提出API (`POST /api/v1/evaluation/feedback-sessions`) が実装済みであること

\- UI ガイドライン（Teal #218F8D）および共通コンポーネント（Textarea, Select, Button）が利用可能であること



\## 実装内容



\### UI 実装



\*\*コンポーネント：FeedbackSessionForm.tsx\*\*



\- 機能：自己評価および事前アンケートの入力・送信

\- 状態管理：フォーム入力値（Controlled Components）、送信ステータス（Draft/Submitting/Completed）

\- 入力項目：

&nbsp; - \*\*評価者選択\*\*: マネージャーを選択（デフォルトは直属の上司）。

&nbsp; - \*\*振り返りセクション\*\*:

&nbsp;   - 「今月達成できたこと」（自由記述）

&nbsp;   - 「直面している課題」（自由記述）

&nbsp;   - 「体調・モチベーション」（5段階スライダーまたは選択式）

&nbsp; - \*\*アクション\*\*:

&nbsp;   - 「下書き保存」（`status: draft` で送信）

&nbsp;   - 「提出する」（`status: completed` で送信、確認ダイアログ表示）



\*\*画面レイアウト\*\*



\- 配置：マイページまたは「評価・1on1」セクションのメインコンテンツ

\- デザイン：

&nbsp; - 圧迫感を与えないよう、余白を広めに取ったシングルカラムレイアウト。

&nbsp; - 質問文は柔らかい口調（例：「課題はありますか？」ではなく「困っていることはないですか？」）を採用。

\## チェックリスト



実装完了時に、以下をすべて確認：



\- \[ ] `FeedbackSessionForm.tsx` が実装され、各入力項目が動作すること

\- \[ ] 「下書き保存」ボタン押下時に、ステータス `draft` でAPIが呼ばれること

\- \[ ] 「提出する」ボタン押下時に、確認ダイアログを経て `completed` でAPIが呼ばれること

\- \[ ] 必須項目のバリデーション（未入力チェック）が機能すること

\- \[ ] 送信中（Loading）にボタンが無効化され、スピナーが表示されること

\- \[ ] 送信完了後に完了メッセージ（Toast）が表示され、一覧画面等へ遷移すること

\- \[ ] スマートフォンでも入力しやすいレイアウト（文字サイズ、余白）になっていること



\## 補足・注意事項



\- ユーザーが長時間入力することを想定し、ブラウザを閉じてもデータが消えないよう `localStorage` 等を用いた簡易的な自動保存機能の実装を検討してください（MVPでは必須ではありません）。

\- ワーディング（言葉選び）は `05\_UI\_UX\_GUIDELINES.md` の「Gentle Professionalism」に従い、評価される側の心理的負担を軽減する表現を使用してください。
